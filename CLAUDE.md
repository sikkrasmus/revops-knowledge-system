## The navigation trio

Three files at the repo root work together. **Read `index.md` before drilling into pages** ‚Äî it's your retrieval starting point.

- [`CLAUDE.md`](CLAUDE.md) ‚Äî this file. Routing map, hand-written.
- [`index.md`](index.md) ‚Äî catalog of every page with one-line summaries. **Generated by `ci reindex` from frontmatter. Never hand-edit. Never LLM-edit.** CI fails the build if it's stale.
- [`log.md`](log.md) ‚Äî append-only audit trail. Events: `ingest`, `auto-merge`, `lint`, `pr-merged`, `pr-opened`, `rollback`, `drift-detected`, `verification-refreshed`. **Automation-appended only.** Use `ci log` to read it filtered.

## Retrieval protocol

When a user asks a question that needs context from this corpus:

1. **Read [`index.md`](index.md) first** to locate candidate pages.
2. For on-machine retrieval, prefer `ci` over native Read/Grep when it fits. Examples:
   - `ci list --type icp --motion sales-assisted`
   - `ci search "discount guardrail mid-market"`
   - `ci show shared/pricing/price-book`
   - `ci sources shared/icp/tier-1` ‚Äî what sources back the claims on the page
3. Fall back to native Read/Grep on `shared/` and `functions/` if `ci` isn't installed or the query doesn't fit the CLI's shape.
4. Cite claim-level sources when summarizing. Every factual claim on a page comes with a `[source: ...]` citation ‚Äî surface these.

## Conventions

- **Frontmatter.** Every page has YAML frontmatter with at least `type`, `title`, `description`, `owner`, `last_reviewed`. Shared + function pages additionally require `verified_until` and `reviewers`. Spec: [`schema/frontmatter.md`](schema/frontmatter.md). Enum values: [`schema/enums.yaml`](schema/enums.yaml).
- **Motion and segment tags.** `motion` and `segment` are frontmatter lists. Allowed values for motion: `plg`, `sales-assisted`, `enterprise`. For segment: `smb`, `mid-market`, `enterprise`.
- **Section labels for motion/segment variants.** Within a page, headings can scope content: `## Enterprise pricing  {motion: enterprise, segment: enterprise}`. Grammar: [`schema/section-labels.md`](schema/section-labels.md).
- **Cross-references.** Use relative markdown links. `ci verify` fails on broken internal links.
- **Slugs.** A page's slug is its repo-relative path without `.md`. Examples: `shared/icp/tier-1`, `functions/sales/README`.
- **Citations.** Every factual claim carries `[source: <ref>]` inline, and the reference must appear in the page's frontmatter `sources:` list.

## CLI shortcuts

When to use `ci` vs. native tools:

- `ci list [filters]` ‚Äî find pages matching frontmatter filters (faster than grep + yq dance)
- `ci search "<query>"` ‚Äî ranked grep (title match + frontmatter boost + recency); prefer over raw grep for cross-corpus queries
- `ci show <slug>` ‚Äî print a page with resolved cross-references; prefer over Read when you want resolved links
- `ci sources <slug>` ‚Äî list source files backing a page's claims
- `ci graph <slug>` ‚Äî inbound + outbound links for a page
- `ci verify <slug>` or `ci verify --all` ‚Äî schema + link + citation checks
- `ci stale` ‚Äî corpus-wide freshness report (`verified_until` passed, `source_drift` flagged, contradictions)
- `ci reindex` ‚Äî regenerate `index.md` from frontmatter. Run locally if you've added pages; CI enforces on every push.
- `ci log --since 7d --type ingest` ‚Äî filtered read of `log.md`
- `ci pending` ‚Äî list open PRs (thin `gh` wrapper)
- `ci diff <pr>` ‚Äî show pending changes with reviewer routing

## Editing rules (abbreviated)

Full spec: [`schema/authoring-contract.md`](schema/authoring-contract.md). Highlights:

- **Never edit `index.md` or `log.md` directly.** Both are automation-maintained. Instead: write a good frontmatter `description` field; `ci reindex` will regenerate the catalog.
- **Claim-level provenance.** Every factual claim cites a source. No unsourced assertions.
- **Changes PR on day one.** Graduated trust stages expand auto-merge later. Some content types (ICP, pricing, packaging, legal, strategic positioning, new page, delete page) always PR.
- **Don't rename or delete pages without an issue.** Page identity is load-bearing ‚Äî breaking slugs breaks links across the corpus.
