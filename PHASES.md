# Phases

Honest roadmap. What's built today, what's next, what's deferred, and what's out of scope.

Each phase states: what's in, what's out, success criteria.

---

## Phase 0 — Shared spine (this release)

**Goal.** Ship a substantive framework that delivers value on day one without overcommitting to any function. Make the seven shared components real, the consumption surface real, and the governance rules real.

**In scope.**

- Seven shared components in `shared/`: `icp/`, `personas/`, `product-and-packaging/`, `pricing/`, `positioning/`, `competitive-intel/`, `data-definitions/`. Each with a README, templates, and at least one abstract example with valid frontmatter.
- Schema layer: `schema/authoring-contract.md`, `schema/frontmatter.md` (base + motion/segment + drift fields), `schema/section-labels.md`, `schema/enums.yaml`, `schema/page-templates/shared/*`.
- Navigation trio: `CLAUDE.md` (hand-written), `index.md` (generated by `ci reindex`), `log.md` (appended by CI on merge).
- CLI (`consumption/cli/ci`, Bash + `yq`): all 11 commands operable.
- `consumption/SKILL.md` — Claude Code / Cowork skill definition.
- `consumption/usage-patterns.md` — worked narratives for AE / Head of RevOps / HITL agent.
- `sources/` scaffolds + `sources/POLICY.md` (hybrid in-repo/out-of-repo).
- `ingestion/`: PR workflow with graduated trust, artifact-commit path, drift-detection lint spec, `auto-merge-rules.yaml` as concept (not wired in v1), abstract `connectors/PATTERN.md`.
- CI: `.github/workflows/verify.yml` (`ci reindex --check` + `ci verify --all`) and `log-append.yml` (appends to `log.md` on merge).
- `functions/` scaffolded with READMEs pointing to phase status. Empty beyond README.

**Out of scope (deferred).**

- Real connector implementations (Confluence, Gong, HubSpot, etc.) — future / community.
- Filling `functions/sales/` content beyond a scaffolded README — Phase 1.
- Conversation-harvest pipeline implementation — Phase 3.
- Auto-merge enforcement (the YAML ships as concept) — Stage 2 operator decision.
- Confidence scoring algorithm — heuristic only in v1, real spec in v2.
- Embeddings / vector DB — never (per principle 4 / 5).

**Success criteria.** All nine items in `SPEC.md` §10 pass:

1. Seven shared components each have README + at least one templated file with valid frontmatter + entry in `index.md`.
2. `ci` runs all 11 commands without error against the v1 corpus.
3. `ci verify --all` returns exit code 0.
4. `ci reindex --check` returns exit code 0 (committed `index.md` matches generation).
5. `verify.yml` and `log-append.yml` are present and CI is green.
6. v0 files in the survival map are migrated or deleted per spec; no orphans.
7. README, CLAUDE.md, PRINCIPLES.md, PHASES.md are written and internally consistent.
8. `SKILL.md` is in place; manual smoke test returns the right page.
9. PR description summarizes Phase 0 scope and links REFRAME / OUTLINE / SPEC.

## Phase 1 — Sales function — shipped

**Goal.** Fill one vertical function end-to-end to prove the pattern generalizes.

**Why Sales first.** Audience demand lives there. v0's existing templates (battlecard, objection, postmortem, champion profile) salvage into `schema/page-templates/sales/`. Sales has the densest downstream references to the Phase 0 shared components (ICP, pricing, personas, positioning, competitive-intel).

**In scope (delivered).**

- `functions/sales/methodology/` — MEDDPICC with motion and segment variants via in-file section labels.
- `functions/sales/playbooks/` — seven deal-stage playbooks: discovery, demo, evaluation, proposal, negotiation, close, expansion-handoff.
- `functions/sales/frameworks/` — three decision frameworks: qualification, deal-desk-escalation, forecast-methodology.
- `functions/sales/plays/` — four plays: outbound-sequence-mid-market, competitive-displacement, referral-ask, renewal-acceleration.
- `functions/sales/battlecards/` — README + one worked example battlecard against an abstract enterprise-incumbent archetype.
- `functions/sales/objections/` — README + one worked example objection ("we can build this internally").
- `functions/sales/tools-and-stack/` — curated tool-category map (CRM, sequencer, call recorder, CPQ, deal desk, forecasting).
- Upgraded templates in `schema/page-templates/sales/`: battlecard, objection, pricing-pattern, postmortem, champion-profile, icp-sales (all six brought to the Phase 0 template standard).
- Shared-spine additions surfaced by Sales fill: `shared/personas/personas/vp-sales.md`, `shared/personas/buying-committees/enterprise-direct.md`.

**Out of scope.** Other functions. Real connector implementations.

**Success criteria — all met.** Pages exist; `ci verify --all` returns 0 fail / 0 warn; `ci reindex --check` current; smoke tests 15/15; every sales page cites at least one `shared/` component in its Related section.

## Phase 2 — Remaining functions — next

**Goal.** Prove the pattern generalizes across all six functions.

**In scope.** Fill `functions/marketing/`, `functions/customer-success/`, `functions/support/`, `functions/revops/`, `functions/finance-legal/` to the same pattern as Sales.

**Ordering.** Sequence by reuse of Phase 0 + Phase 1: Marketing next (shares positioning, competitive-intel, ICP heavily), then CS (shares ICP, personas, pricing for expansion), then RevOps (owns data-definitions), then Finance/Legal (authored-shared: pricing, contracts, compliance posture), then Support.

**Out of scope.** Real connector implementations.

**Success criteria.** Each function directory has methodology, playbooks, frameworks, plays. Cross-function references resolve. `ci verify --all` passes. `ci graph` shows dense cross-function citation patterns.

## Phase 3+ — Advanced ingestion & maintenance

**Goal.** Reduce the HITL burden at scale.

**In scope.**

- **Conversation-harvest pipeline.** Ingest Claude Code / Cowork session transcripts from team members; extract candidate changes; open PRs with diffs, provenance, and confidence. PII filtering, consent, cross-session deduplication, confidence scoring, author attribution.
- **Automated staleness detection with LLM.** Beyond `verified_until` dates and lint regex — LLM passes that detect subtler contradictions and deprecated language.
- **Cross-corpus consistency checks.** Compare pages that reference each other for factual consistency; flag divergences.
- **Stage 2 → Stage 3 auto-merge enforcement.** Wire `ingestion/auto-merge-rules.yaml` into CI with configurable per-stage rules.

**Out of scope for v1 of Phase 3.** Federated Company Indexes across multiple companies (that's a different product).

**Success criteria.** The HITL burden measurably drops between Stage 2 and Stage 3. Rollback incidents stay below a threshold. The log shows the automated paths are working.

## What's never in scope

- **Embeddings or a vector DB.** Per principle 4 / 5. If scale demands better retrieval, the answer is better structure (sub-indexes), not a vector engine.
- **A CMS or database-backed storage.** The corpus is markdown in git. Period.
- **Multi-tenant SaaS hosting.** This framework is self-hosted. Someone will productize it; that's their project, not this one.
- **Hard gates on which functions you adopt.** Adopt the shared spine only, or only Sales, or only RevOps — each is a valid slice.

---

## When phases unlock

Phase 1 shipped in April 2026 on top of the Phase 0 merged baseline. Phase 2 unlocks when Phase 1 is stable in practice and Sales is in production for at least one pilot. Phase 3 unlocks when Stage 2 governance is wired in practice (not just on paper).

No fixed dates. Dates would be wrong.
