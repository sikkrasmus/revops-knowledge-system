## Why grep-native, no embeddings

See [`../PRINCIPLES.md`](../PRINCIPLES.md) §5. TL;DR: at the scale a single company's KB reaches, grep + frontmatter + the navigation trio beats a vector DB on cost, transparency, portability, and debuggability. When scale breaks grep, the answer is better structure (sub-indexes, function-level indexes), not embeddings.

## The three pillars of retrieval

1. **[`../index.md`](../index.md)** — catalog, read first. Generated by `ci reindex`.
2. **[`../CLAUDE.md`](../CLAUDE.md)** — routing, hand-written. Loaded on session start.
3. **`ci` CLI** — deterministic queries. Filters, grep, graph, freshness.

Any agent can substitute native Read/Grep for the CLI when the CLI isn't installed. The CLI is a convenience wrapper, not a dependency.
